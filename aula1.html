CAPÍTULO 01

A Máquina e o Intérprete

Computadores são máquinas de processar sinais elétricos. Eles não entendem "Olá". Eles entendem 0 e 1 (Binário).

A programação é a arte de traduzir o pensamento humano para algo que a máquina processe. Linguagens como JavaScript servem como essa ponte.

CAPÍTULO 02

Variáveis: Memória seletiva

Tudo em um sistema precisa ser guardado. Nome de usuário, saldo, nível de bateria. Usamos variáveis para nomear espaços na memória RAM do computador.

// let: usado para valores que podem mudar
let saldo = 1500.00;
saldo = 2000.00;

// const: usado para valores que nunca mudam
const gravidade = 9.8;
const nomeApp = "JS Academy";
Dica Pro: Sempre use const por padrão. Só use let se você tiver certeza que o valor vai mudar. Isso evita erros de lógica no futuro.
CAPÍTULO 03

O Fluxo de Decisão (IF/ELSE)

Um programa sem decisões é apenas uma lista de tarefas. A verdadeira inteligência nasce quando o código escolhe caminhos baseados em condições.

let bateria = 9; // Sua bateria atual

if (bateria < 10) {
    console.log("Modo Economia: ATIVADO");
} else {
    console.log("Uso Normal: ATIVADO");
}

Aqui usamos operadores de comparação: < (menor), > (maior), === (igual) e !== (diferente).

CAPÍTULO 04

Funções: O Poder da Repetição

Imagine ter que escrever 20 linhas de código toda vez que um usuário clicasse em "Comprar". Seria exaustivo. Funções permitem que você empacote uma lógica e a use quantas vezes quiser.

function enviarMensagem(usuario) {
    return "Olá, " + usuario + "! Bem-vindo à MineDevs.";
}

// Chamando a função
console.log(enviarMensagem("Você"));
